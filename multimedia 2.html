..::HISTORIA DE GIT::..

Cuando GitHub inició en el 2008, todo era compartir. Una persona podía registrarse rápidamente para obtener una cuenta y compartir su código abierto con el mundo, o comprar repositorios privados y controlar con precisión quién tenía acceso a su código fuente. Pero el objetivo no era flujo de trabajo o colaboración,si no facilitar compartir repositorios Git con los demás. A medida que GitHub creció se hizo más evidente el poder de trabajar en conjunto. Esto llevó a crear Organizations en el 2010: cuentas de grupo que permiten que los proyectos de código abierto, sin fines de lucro -escuelas, gobiernos, compañías, y equipos de todo tipos- creen presencia y construyan fácilmente software juntos. Su enfoque se expandió, de simplemente publicar repositorios Git, a ayudar a las personas construir software en equipo.
